異世界転生ジャンルの小説企画を作成してください。

日本国内だけでなく世界的なトレンドも踏まえ、今後10年間で大きくバズる可能性が高い
作品案を30本提案してください。

# 要件
- 30本分の「タイトル + あらすじ」を作成すること
- 1つのテキストファイルに30本すべてをまとめること
- `sample.txt` は「フォーマットのみ」参考にすること
- `sample.txt` のタイトル内容・あらすじ内容は参考にしないこと

# 出力フォーマット（各作品共通）
```
# タイトル名

## あらすじ
（2000文字程度）
```

# 保存ルール
- 作品本文は `story_master.txt` の1ファイルに追記保存すること（新規ファイルを増やさない）
- 追記時は、生成単位ごとに `batch_id`（UTCミリ秒）を見出しとして付与すること
- 例: `batch_id: 1767179917572`

# 重複回避ルール（重要）
- 過去の本文を毎回すべて読み直すのではなく、要約インデックスを参照すること
- 要約インデックスファイル名は `story_index.tsv` とすること（1ファイルで蓄積管理）
- 新規30本を作成する前に、`story_index.tsv` の全行を確認し、既存案と内容が重複しない企画のみ作ること
- 新規作成後、30本すべてについて要約インデックスを追記更新すること

# 要約インデックス仕様（story_index.tsv）
- 1行 = 1作品
- タブ区切りで以下の列を使用すること

```
batch_id\ttitle\tsummary_100\tgenre_tags\tcore_hook
```

- `batch_id`: 生成バッチID（UTCミリ秒、例: 1767179917572）
- `title`: 作品タイトル
- `summary_100`: あらすじを約100文字に圧縮した要約
- `genre_tags`: 主要タグを3〜5個（例: 転生, 学園, 復讐, 経営）
- `core_hook`: その作品の中核ギミックを1文で記述

# 重複判定基準
- 以下が既存行と実質的に同じなら重複とみなし、別案に差し替えること
	- 主人公の初期条件
	- 世界設定の中核ルール
	- 物語の勝ち筋（問題解決の軸）
	- コアフック（読者が最初に引かれる仕掛け）
- タイトルだけ変えた案、職業や固有名詞だけ変えた案は重複扱いにすること
